# Contributing Guide

Terima kasih telah tertarik untuk berkontribusi pada Web Management Sawit!

## Cara Berkontribusi

### Reporting Bugs

Jika Anda menemukan bug:
1. Periksa apakah bug sudah dilaporkan di Issues
2. Buat issue baru dengan label "bug"
3. Jelaskan bug secara detail
4. Sertakan langkah-langkah reproduksi
5. Sertakan screenshots jika ada
6. Sertakan informasi environment (OS, browser, dll)

### Suggesting Features

Jika Anda ingin menyarankan fitur baru:
1. Periksa apakah fitur sudah disarankan di Issues
2. Buat issue baru dengan label "enhancement"
3. Jelaskan fitur secara detail
4. Jelaskan kegunaan fitur
5. Sertakan mockup jika ada

### Pull Requests

1. Fork repository
2. Buat branch untuk fitur/bug fix
3. Commit perubahan Anda
4. Push ke branch Anda
5. Buat Pull Request

#### Pull Request Checklist

- [ ] Code mengikuti style guide
- [ ] Self-review code
- [ ] Comment code yang kompleks
- [ ] Update documentation jika perlu
- [ ] Test perubahan Anda
- [ ] Tidak ada breaking changes (atau didokumentasikan)
- [ ] Error handling ditambahkan
- [ ] Code tested dan berfungsi

## Code Style

Ikuti style guide yang ada di DEVELOPMENT.md

## Testing

Sebelum submit PR:
- Pastikan semua test pass
- Test fitur baru yang ditambahkan
- Test bug fix yang diperbaiki

## Questions?

Jika ada pertanyaan, silakan buat issue dengan label "question".
